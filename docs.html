RESPONSIVE REDESIGN Original Site Redesigned Site

00 Intro

The responsive redesign was a project spanning a few weeks, which involved redesigning a preexisting webpage to adhere more closely to both design principles and accessibility guidelines. During this timeline, I worked on designing lofi and hifi wireframes, creating a style guide, and developing the page with HTML and CSS.

01 Identifying Usability Problems

For this project, I chose the website for TP TEA, which is my favorite boba store back at home; even though the user interface is already pretty well-developed, I think it would be an interesting challenge to further improve the webpage, particularly in bringing a more technical and user-driven focus to a website that currently seems to be more storytelling and aesthetic-driven.

02 Problems

01 Efficiency

One issue with efficiency comes from the interactive components, primarily the navigation bar. Upon entering the site, the bar is hidden and doesn’t appear until the user starts scrolling, which can be inefficient given the convention to either have the navigation already displayed or to have it display on hovering near the top of the site.

Even if frequent users know they must scroll down to access the navigation bar, doing so still costs extra time, which can slow them down when they are trying to access other parts of the site.

02 Learnability

The biggest issue with learnability in this website comes from the language barrier, given that some elements (like the pop-up in the bottom left corner) are still written in Chinese despite being on the English version of the site.

Some animations and transitions on the homepage are inconsistent as the user goes down the page, with some elements transitioning after others in the same group have already loaded. With this inconsistency, it becomes difficult to know when to wait for things to load and when to know everything has already been displayed, which can lead to a less smooth user experience.

03 Memorability

The homepage of the website is pretty long since it contains an overview of each section, which can also slow users down if they’re trying to get to a specific area of the site via the sections on the homepage; to do so, they must read through each section to figure out where the button to their desired destination is located, especially because the placement of buttons is often inconsistent.

As a casual user, it might be easy to remember that links to the other pages of the site can all be found on the homepage, yet difficult to remember where those links are placed.

03 Accessibility

Using WebAIM WAVE, a few recurring problems with the website surfaced as errors: there were three images missing alternative text, seven empty links, and three broken ARIA references. In addition to these 14 errors were a host of 42 alerts, which included problems with redundant links or title text, skipped header levels, justified text, and suspicious link text.

Although I agree with some of the issues, like missing alt text, empty links, and justified text, I didn’t agree with other alerts concerning redundant links/title text and skipped header levels. For redundant links/title text, it seemed like WebAIM WAVE was mostly focused on individual words rather than the entire text itself. Additionally, for skipped header levels, I didn’t feel that it was always necessary to go in order, since having three header levels like an H1, H2, and H3 can impede visual hierarchy given their similarities in size.

04 Speed Sketching

For my sketches, I focused on both the structure of the page and on working through sections where I felt visual hierarchy was a bit more disorganized.

05 Final Sketch

The final sketch combined a few elements from the initial sketches, with a simplified page structure and modular components. Although it stayed pretty loyal to the original site, it also reorganized the information in a way that was more consistent and readable.

09 Responsive Redesign

The FINAL REDESIGN implements all of the hifi designs from before, with some minor tweaks to spacing to adapt to real-life sizing and modularity of CSS components.

The only issues that surfaced in WebAIM were ones concerning repetitive alt text, but I wanted to keep the content of the alt text related to the drinks featured in the Menu section; however, given that all of these are drinks (many of them milk teas), WebAIM likely found issue with the duplicate labels.

10 Notes

Overall, it was a pretty fun experience redesigning and coding a website that I was somewhat familiar with before. Even though I have had a decent amount of frontend and UI/UX experience, I still felt like I was able to learn some more CSS techniques that I hadn't known before, and seeing this website through a more accessibility-focused lens rather than one that was more graphic design-focused was also interesting as well.
